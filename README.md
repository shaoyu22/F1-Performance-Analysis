# 🏎️ F1 Performance & Telemetry Analysis: Suzuka 2024

## 📌 Project Overview
As a fan of Formula 1 and a data enthusiast, I initiated this project to explore the intersection of **human performance** and **automotive engineering**. This project specifically analyzes the 2024 Japanese GP at Suzuka to uncover driver technicality and performance tiers.

## 🛠️ Tech Stack
- **Language:** Python
- **Data Source:** [FastF1](https://github.com/theOehrly/FastF1) / [OpenF1 API](https://openf1.org/)
- **Libraries:** Pandas, Matplotlib, Seaborn, Requests
- **Key Focus:** Data Cleaning, Telemetry Visualization, Performance Metrics

## 📂 Project Structure
- **/data**: Raw and processed race telemetry and lap data.
- **/notebooks**: Phase-based exploratory data analysis (EDA).
- **/scripts**: Python automation scripts for data fetching and API integration.
- **/visualizations**: High-level performance charts and tiering insights.

---

## 🏁 Project Roadmap

### 📊 Phase A: Sector 1 Technical Mastery (S-Curves) - [Completed]
* **Objective:** Evaluate driver consistency through high-speed technical corners.
* **Key Insight:** **Carlos Sainz (#55)** demonstrated superior technicality in Sector 1, maintaining an average pace ~0.2s faster than the immediate competition.
* **Visualization:** `visualizations/sector1_tiering.png`

### 🌦️ Phase B: Weather Data Integration - [In Progress]
* **Objective:** Integrate **Open-Meteo API** to fetch real-time track temperature and precipitation.
* **Focus:** Analyzing how track dampness affects traction and cornering speeds in the S-Curves.

### 🏁 Phase C: Strategy Impact & Driver Tiering - [Planned]
* **Objective:** Combine telemetry and weather variables to build a "Performance Resilience Model."
* **Goal:** Evaluate how different tyre compounds and weather shifts affect lap time degradation.
